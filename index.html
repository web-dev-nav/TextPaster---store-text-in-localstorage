The app allows users to enter a heading and associated text. This way, users can add important data and have it later on. Some use cases are: storing login information, notes, etc. When the "Save Text" button is clicked, the entered text is saved to the browser's local storage and retain their even after page refresh and never expired until you delete or clear cookies. Moreover, If the heading already exists, the existing entry is updated; otherwise, a new entry is added.
Overall, the Text Storage App is a simple productivity tool that allows users to save, edit, and download their text entries with headings, facilitating better organization and access to important information.

The page with a title, text input, and textarea for entering the heading and text. There are two buttons for saving the text and refreshing the list of saved texts.

The app provides a "Download" button, which, when clicked, generates a .txt file containing all saved texts along with their headings. Users can download this file.

The saved texts are displayed on the side of the page in a list format. Each saved text entry shows a truncated version of the text with a "Show More" link to view the full text.

If there are more than five saved text entries, pagination is added to display the saved texts in multiple pages.

For each saved text entry, an "Edit" button is provided. When clicked, the heading and text fields are populated with the saved text, allowing users to edit the text. Users can click the "Save Text" button after editing to update the existing entry.

For each saved text entry, a "Delete" button is provided. When clicked, the corresponding entry is deleted from the saved texts list.

We used Bootstrap to style the app and ensure consistent design across all pages. We also added Font Awesome icons to the buttons to enhance the user interface.

The app checks for empty text inputs before saving and displays appropriate error messages. If no saved texts are found, a message is displayed indicating that there are no saved texts.

If there are no saved texts, the pagination is hidden to avoid unnecessary UI elements when there are no saved entries.

For each saved text entry, a specific timestamp is added to let users remember the date and time they posted the text.

Both pages are capable to view on tablet and mobile device.
